github+hexo收集的零碎资料

发表于 2018-03-27 | 分类于 零碎资料 |

零碎资料

hexo常用语句

  1. hexo generate 生成静态文件，也就是之后部署在GitHub Pages上的文件。
  2. hexo server 启动服务器。访问网址localhost:4000可以在本地预览博客的内容和排版。同时这个命令最开始已经使用过。
  3. hexo deploy 部署网站。将网站页面代码部署在GitHub上，之后就可以通过your_username.github.io访问了。
  4. hexo clean 清除缓存文件和已生成的静态文件.在某些情况（尤其是更换主题后），如果发现站点的更改无论如何也不生效，可能需要运行该命令。

    一般使用步骤

  5. 通过 hexo new命令产生一个新的 xxxx.md文件。(也可以在目标目录下自己新建文件)
  6. 在xxxx.md完成相关的内容书写。
  7. 通过命令hexo s开启本地服务，通过浏览器访问http://localhost:4000预览效果。
  8. 确认无误后通过hexo g生成页面文件。
  9. 通过命令hexo d部署到GitHub上。

    leancloud数据管理

    访问量 https://leancloud.cn/dashboard/applist.html#/apps

    注意事项

  10. Hexo d之前先输一下代码 npm install –save hexo-deployer-git

  11. Name.github.io name=自己github的id
